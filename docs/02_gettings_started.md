# How a user gets started

There are multiple potential starting points for a user:

## Start with a "wireframe"

A web designer starts a new project and wants to begin with a wireframe. They will have multiple options to "import" their wireframe.

1) Upload the raw HTML of a website or wireframe
2) Paste a URL (maybe an existing website)
3) Upload a JPG/PDF/PNG of a wireframe or designed website
4) Use a figma MCP to import from a figma design
5) Use AI to directly "vibe code" the wireframe layout

Our AI will convert into an editable wireframe in OUR specified design system. Once the wireframe is created in the app, it will then automatically (or through a button) write the copy out to the raw editable document for you to play with and edit. Sections in the document will map to sections in the wireframe. Text will be turned into H1s, H2s, CTAs, paragraphs, lists, etc, per the wireframe. 

Each section will be editable and versioned. Users can create and propose new versions of the copy, and see the changes between their version and the "chosen" version.

## Start with the "copy"

Users may want to freeform start writing the website copy itself, before the wireframe. They want a blank page that acts like a google doc/notion page. They write messy free flowing copy that slowly start to structure into H1s and paragraphs and CTA buttons as they go. They are organizing their copy by sections. And even making multiple versions of any given section that they can toggle through to see what their favorite is. No wireframe may exist yet. When they are done writing, they'll convert the copy into a wireframe (per our design system) so they can visually see how their text looks.

## Start with an existing site

You may be a copywriter or web designer that is working on making edits to an existing site. You can paste a site URL or the raw HTML, and the app will populate both the raw copy in the editable text document and the wireframe with content from the website. Sections from the copy are automatically linked to sections in the wireframe. This represents using our app mid process, or coming back to the app after you've done some revisions with the client.
